Data Flow

Project Execution Flow

User clicks "Run" | v ProjectDetail.tsx --invoke("run_project")--> commands.rs::run_project() | +--------+--------+ | | store.get(uuid) registry.lock() | (prevent double-run) v executor::spawn_and_stream() | Returns (Child, Receiver<LogLine>) | +----------+-----------+ | | Child stored in tokio::spawn(log_task) ProcessRegistry | reads from Receiver | app.emit("project-log", LogEvent) | v ProjectDetail.tsx listen("project-log") appends to logs state renders in log viewer When process exits: log_task detects channel closed --> registry.remove(uuid) --> child.wait() for exit code --> store.update_status(Idle or Failed) --> app.emit("project-status-change")

Project Creation Flow

PasteAndDeploy.tsx | +--> invoke("detect_runtime", { path }) | | | runtime::detect_runtime() | | | Scans for marker files: | requirements.txt -> Python | package.json -> Node | go.mod -> Go | Cargo.toml -> Rust | | | Returns RuntimeInfo { runtime, entrypoint, confidence } | +--> invoke("create_project", { name, path }) | commands.rs::create_project() | detect_runtime + Project::new + store.insert | Returns Project (saved to SQLite)

    Running,    // Process active
    Stopped,    // Manually stopped
    Failed,     // Non-zero exit code
}

MCP Server (Phase 2)

12 tools defined for AI agent control via Model Context Protocol:

ToolPurpose
runway_list_projectsList all projects with status
runway_project_statusDetailed status of one project
runway_deployDeploy code to a target
runway_stopStop a running deployment
runway_restartRestart a deployment
runway_logsStream or fetch logs
runway_list_targetsList configured deploy targets
runway_add_targetConfigure a new target
runway_list_agentsList connected agents
runway_import_codeImport code from path or stdin
runway_detect_runtimeAuto-detect language and deps
runway_healthSystem health check

Transport: stdio (primary, for Claude Code) + HTTP via axum (for remote/web clients).

CLI (Phase 2)

runway list                                    # List projects
runway deploy <path> --target <target>         # Deploy code
runway logs <project> [-f|--follow]            # View logs
runway status <project>                        # Check status
runway targets list                            # List deploy targets
runway targets add <name> --host <host>       # Add target
